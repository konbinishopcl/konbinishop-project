# Phase 0 · Plan 00-01: Re-alineación GSD

**One-liner:** Re-alinear toda la documentación de planning (`.planning/`) con la realidad
del proyecto tras la migración de stack Strapi/Nuxt → NestJS/Next.js y el cambio de modelo a
publicación gratuita de eventos.

## Contexto

El planning vigente (creado el 2026-03-23) describía un stack y un producto que ya no
existen:

- **Stack obsoleto:** Strapi 5 + Nuxt 4 + dashboard Next.js → hoy NestJS 11 + Prisma 6 +
  una sola app Next.js (sitio público + admin).
- **Modelo obsoleto:** milestone de pagos (organizador paga por publicar; pasarelas
  Transbank / Mercado Pago / Stripe; emails MJML). Hoy: publicar es **gratis** en v1, no se
  venden entradas (la venta ocurre en una plataforma externa), el cobro al organizador se
  difiere a v2.

Planear cualquier fase de feature sobre esa base produciría trabajo equivocado, por lo que
la re-alineación es prerequisito de todo lo demás.

## Decisión del usuario

- Foco de la fase: **re-alinear el roadmap** (de 4 opciones presentadas).
- Modelo de negocio: **gratis por ahora, pagos después** — v1 es publicación gratuita; el
  cobro al organizador queda como milestone v2.
