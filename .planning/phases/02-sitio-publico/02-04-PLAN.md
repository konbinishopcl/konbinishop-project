# Phase 2 · Plan 02-04: Quitar el checkout + cerrar Phase 2

**One-liner:** Eliminar del sitio el flujo de checkout/venta de entradas (error de diseño,
events-only) y conectar el `HeroBlock` a eventos reales — con esto el sitio público queda
100% con datos de la API.

## Contexto

Konbini no vende entradas (la venta es en una plataforma externa). La maqueta arrastra una
ruta `/checkout/[id]` y el `HeroBlock` aún enlaza a `/checkout/1` con datos mock. Este plan
elimina ese resto y deja el hero con datos reales — último paso de Phase 2.

## Tareas

1. **Eliminar** la ruta `(site)/checkout/[id]/`.
2. **`lib/api.ts`** — agregar el tipo `HeroEvent` y el mapper `toHeroEvent(ApiEvent)`.
3. **`HeroBlock.tsx`** — recibir `events: HeroEvent[]` por prop; carrusel real (flechas y
   dots recorren los destacados); CTA "Ver evento" → `/evento/<slug>` (sin checkout).
4. **`(site)/page.tsx`** — construir los `heroEvents` desde los eventos de la API y pasarlos
   al `HeroBlock`.
5. **Verificar** que no queda ninguna referencia a `/checkout` en el código.

## Decisiones

- La ruta de checkout se elimina por completo — la compra de entradas ocurre fuera del sitio.
- El `HeroBlock` deja de usar el mock `HERO`; muestra los primeros eventos reales como
  carrusel destacado.
- El `HERO` mock en `lib/data.ts` queda sin uso (se limpiará al retirar el resto del mock).

## Verificación

- `pnpm build` del website compila sin errores
- No existe la ruta `/checkout`; `grep` no encuentra referencias a `/checkout`
- El `HeroBlock` muestra un evento real y el CTA enlaza a `/evento/<slug>`
- Cierra Phase 2: home, categorías y detalle del sitio público corren con datos reales
