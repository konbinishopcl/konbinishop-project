# Phase 3 · Plan 03-02: Subida de imágenes en /crear

**One-liner:** Habilitar la subida de banner, poster y galería en el Step 3 del formulario
`/crear`, contra `POST /api/upload`.

## Contexto

03-01 dejó el formulario creando eventos sin imágenes (Step 3 mostraba "disponible pronto").
La API ya tiene `POST /api/upload` (Phase 1, plan 01-03). Este plan lo conecta.

## Tareas

1. **`lib/api.ts`** — `api.uploadImage(file, token)` — `POST /api/upload` multipart
   (`FormData`, sin `Content-Type` manual); devuelve `{ url }`.
2. **`/crear/page.tsx`**:
   - `FormData` gana `banner`, `poster` (string) y `gallery` (string[]).
   - Componente `ImageUploader` reutilizable: file picker oculto, sube al elegir, muestra el
     preview y permite quitar la imagen.
   - Step 3: banner y poster con `ImageUploader`; galería con thumbnails subidas + un botón
     "Agregar" (hasta 10).
   - El submit incluye `banner` / `poster` / `gallery` en el `CreateEventInput`.

## Decisiones

- La subida es inmediata al elegir el archivo (no diferida al submit): cada imagen va a
  `/api/upload` y el formulario guarda la ruta `/uploads/...` devuelta.
- Validación de tipo/tamaño la hace la API (JPG/PNG/WebP, 5 MB); el formulario muestra el
  error que devuelve.

## Verificación

- `pnpm build` del website compila sin errores
- `POST /api/upload` con una imagen devuelve `{ url }` y el archivo queda servido en `/uploads`
- Un evento creado desde `/crear` con imágenes guarda las rutas `/uploads/...` en
  `banner`/`poster`/`gallery`
