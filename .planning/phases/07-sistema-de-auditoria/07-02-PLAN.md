  <action>
    Agregar al final de `apps/api/prisma/schema.prisma` (después del modelo `Like`) una nueva
    sección con dos enums y un modelo. Usar EXACTAMENTE este bloque:

    ```prisma
    // ─────────────────────── Auditoría ───────────────────────
    // Registro de acciones de admins y usuarios sobre las entidades del sistema.
    // userId es Int? SIN relación FK: así un usuario eliminado no borra su historial
    // de auditoría (cumplimiento / trazabilidad). El join con User, si se necesita el
    // email del actor, se hace manualmente en el endpoint de consulta.

    enum AuditAction {
      CREATE
      UPDATE
      DELETE
      APPROVE
      REJECT
      BAN
      UNBAN
    }

    // Nombres comerciales del producto: AVISO = Spot, PORTADA = Hero.
    // El enum usa los nombres de marca por decisión de producto (D-Phase7).
    enum AuditEntity {
      EVENT
      USER
      AVISO
      PORTADA
    }

    model AuditLog {
      id        Int         @id @default(autoincrement())
      userId    Int?        // actor; null si la acción es de sistema o el usuario fue eliminado
      action    AuditAction
      entity    AuditEntity
      entityId  Int
      metadata  Json        @default("{}")
      ip        String?     // dato personal (Ley 21.719) — retención 24 meses
      userAgent String?     // dato personal (Ley 21.719) — retención 24 meses
      url       String?
      createdAt DateTime    @default(now())

      @@index([entity, entityId])
      @@index([userId, createdAt])
      @@index([createdAt])
      @@index([entity, action, createdAt])
      @@map("audit_logs")
    }
    ```

    NOTAS IMPORTANTES:
    - El enum `AuditEntity` usa `AVISO` y `PORTADA` (nombres comerciales) por decisión LOCKED
      del usuario. La investigación (07-RESEARCH.md, Open Question #1) sugería `SPOT`/`HERO`,
      pero la decisión de producto manda: AVISO = Spot, PORTADA = Hero.
    - `userId` es `Int?` SIN `@relation` — no declarar FK a `User`. Esto preserva el historial
      ante el borrado de usuarios.
    - `@@map("audit_logs")` fija el nombre de tabla en snake_case.
    - NO agregar un campo `heroes`/`spots` ni relación inversa en `User` ni en otros modelos.
  </action>
