    **2. `apps/api/src/audit/audit.service.ts`** — `AuditService` singleton (`@Injectable()`,

    Exportar el tipo de parámetros de log:
    ```ts
    import type { Request } from 'express';
    import type { AuditAction, AuditEntity } from '@prisma/client';

    export interface AuditLogParams {
      userId: number | null;
      action: AuditAction;
      entity: AuditEntity;
      entityId: number;
      metadata?: Record<string, unknown>;
      req?: Request;
    }
    ```

    Método `log(params: AuditLogParams): void` — FIRE-AND-FORGET. No es `async`, no se hace
    `await`. Extrae `ip = params.req?.ip ?? null`, `userAgent = params.req?.get('user-agent') ?? null`,
    `url = params.req?.originalUrl ?? null`. Llama a `this.prisma.auditLog.create({ data: {...} })`
    y encadena `.catch(err => this.logger.error('AuditLog insert failed', ...))`. NUNCA debe
    propagar la excepción ni revertir la operación de negocio.

    Método `async findAll(query: QueryAuditDto)` — construir un `Prisma.AuditLogWhereInput`:
    - `entity` si está presente
    - `action` si está presente
    - `userId` si está presente
    - `createdAt: { gte: new Date(dateFrom), lte: new Date(dateTo) }` — incluir solo los
      límites presentes. Para `dateTo`, fijar la hora al final del día (`23:59:59.999`) para
      que el filtro sea inclusivo.
    Paginación: `page = query.page ?? 1`, `pageSize = Math.min(query.pageSize ?? 50, 200)`.
    Usar `this.prisma.$transaction([findMany(...), count(...)])` igual que `EventsService.findAll`.
    Ordenar `orderBy: { createdAt: 'desc' }`. Devolver `{ items, total, page, pageSize, totalPages }`.
