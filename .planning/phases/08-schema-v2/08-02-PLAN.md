<objective>
Reemplazar la jerarquía geográfica de 2 niveles (Region + Commune) por 3 niveles (Country + State + City) en el schema, el seeder y todo el código de aplicación que la consume. Sin este plan, `pnpm tsc --noEmit` queda roto al instante por las 46 referencias documentadas en RESEARCH.md (Pitfall #3).

Purpose: SCH-02 habilita la internacionalización futura (Chile es solo un Country más) y armoniza el catálogo con el shape que el nuevo frontend de v2 espera (`GET /countries`, `GET /states?country=slug`, `GET /cities?state=slug`).

Output: schema.prisma sin Region/Commune; seed.ts con Country "Chile" + 16 States + ~350 Cities; catalog module reescrito; DTOs de eventos con `cityId` en vez de `regionId/communeId`; profiles.service.ts con `city: true` en vez de `region/commune: true`; migración Prisma aplicada.

**Asunción documentada (Open Question #1 — RESEARCH.md):** la base de datos del entorno de desarrollo y del VPS contiene solo datos de seed (no organizadores reales registrados). Por tanto se usa `prisma migrate reset` + `prisma:seed` como estrategia, sin script de data migration regionId→cityId. Esta asunción debe ser CONFIRMADA por el usuario en el Task 0 (checkpoint) antes de ejecutar el reset destructivo del Task 1.
</objective>

    // ─────────────── Geografía: Chile como Country, regiones como States, comunas como Cities ───────────────

    // Limpiar en orden FK-safe: City → State → Country
    await prisma.city.deleteMany();
    await prisma.state.deleteMany();
    await prisma.country.deleteMany();

    // 1. País Chile
    const chile = await prisma.country.upsert({
      where: { slug: 'chile' },
      update: {},
      create: { name: 'Chile', slug: 'chile' },
    });

    // 2. States (16 regiones)
    for (const { region, communes } of regionsData) {
      const stateSlug = slugify(region);
      const state = await prisma.state.upsert({
        where: { slug: stateSlug },
        update: { name: region, countryId: chile.id },
        create: { name: region, slug: stateSlug, countryId: chile.id },
      });

      // 3. Cities (comunas de esa región)
      for (const communeName of communes) {
        const citySlug = slugify(communeName);
        await prisma.city.upsert({
          where: { slug: citySlug },
          update: { name: communeName, stateId: state.id },
          create: { name: communeName, slug: citySlug, stateId: state.id },
        });
      }
    }
