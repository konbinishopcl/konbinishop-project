<objective>
Crear el módulo NestJS `organizations` con CRUD básico sobre el modelo `User` con `type=ORGANIZATION`. La organización es un User sin credenciales (sin password, sin googleId, sin 2FA) cuyo creador queda registrado como OWNER en la tabla `OrgMember` en la misma transacción.

Purpose: Da soporte a ORG-02. Es la base sobre la que se construyen membresías (09-03), transferencias (09-04) e integración con eventos/spots/heroes/orders (09-05). Sin este módulo no hay "organización" a la que invitar miembros o transferir contenido.

Output: Módulo `organizations` registrado en AppModule con 4 endpoints públicos de la API (POST/GET/PATCH/DELETE) y `OrganizationsService` exportado.
</objective>

    1. `async create(dto: CreateOrganizationDto, user: JwtUser, req?: Request): Promise<User>`
       - Usar `prisma.$transaction(async (tx) => { ... })`:
         - `const org = await tx.user.create({ data: { email: dto.email, firstname: dto.name, handle: dto.handle, type: UserType.ORGANIZATION, passwordHash: null, confirmed: true, role: 'AUTHENTICATED' } })`

         - return org
       - Capturar `Prisma.PrismaClientKnownRequestError` code `P2002` y lanzar `ConflictException` con mensaje del campo en conflicto (email o handle).
       - Audit: `this.audit.log({ userId: user.sub, action: 'CREATE', entity: 'USER', entityId: org.id, metadata: { orgCreation: true }, req })` post-transacción.

    2. `async findOne(id: number, user: JwtUser): Promise<User>`
       - `const org = await prisma.user.findUnique({ where: { id }, include: { profile: true, orgMembers: { select: { userId: true, role: true } } } })`
       - Si null o `org.type !== 'ORGANIZATION'` → `NotFoundException('Organización no encontrada')`
       - Si org está blocked y el user no es admin → `NotFoundException`.
       - Si el user NO es admin y NO es miembro → `ForbiddenException('No eres miembro de esta organización')`
         - Para verificar membresía: `await prisma.orgMember.findUnique({ where: { userId_orgId: { userId: user.sub, orgId: id } } })`
       - Devolver org sin `passwordHash`, `twoFactorCode`, `twoFactorExpiry`, `resetToken`, `resetTokenExpiry`.

    3. `async update(id: number, dto: UpdateOrganizationDto, user: JwtUser, req?: Request): Promise<User>`

       - Verificar que el user actual sea OWNER (consulta a OrgMember role===OWNER) o ADMIN/SUPER_ADMIN. Si no → `ForbiddenException`.

    ```typescript
    @Post()
    @UseGuards(JwtAuthGuard)
    create(@Body() dto: CreateOrganizationDto, @CurrentUser() user: JwtUser, @Req() req: Request) {
      return this.service.create(dto, user, req);
    }

    @Get(':id')
    @UseGuards(JwtAuthGuard)
    findOne(@Param('id', ParseIntPipe) id: number, @CurrentUser() user: JwtUser) {
      return this.service.findOne(id, user);
    }

    @Patch(':id')
    @UseGuards(JwtAuthGuard)
    update(@Param('id', ParseIntPipe) id: number, @Body() dto: UpdateOrganizationDto, @CurrentUser() user: JwtUser, @Req() req: Request) {
      return this.service.update(id, dto, user, req);
    }

    @Delete(':id')
    @UseGuards(JwtAuthGuard)
    remove(@Param('id', ParseIntPipe) id: number, @CurrentUser() user: JwtUser, @Req() req: Request) {
      return this.service.remove(id, user, req);
    }
    ```
