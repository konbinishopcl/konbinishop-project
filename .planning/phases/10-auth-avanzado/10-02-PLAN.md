## Goal

Modificar el flujo Google (`POST /auth/google` y `POST /auth/google/onetap`) para distinguir entre usuario **existente** (comportamiento actual: JWT definitivo) y **nuevo** (retorna `onboardingToken` de 30 min con `onboardingPending: true`, sin JWT definitivo). Agregar `POST /auth/google/onboarding` protegido con `OnboardingGuard` que recibe `{ countryId, acceptedTerms }`, actualiza el `User` y emite el JWT definitivo.

## Context

**Dependencia:** Plan 10-01 ya estableció el patrón `sign{Tipo}Pending(user)` + guard correspondiente. Replicarlo para onboarding.

El método `upsertGoogleUser()` (líneas 102-118 de `auth.service.ts`) actualmente retorna solo `user: User`. Debe modificarse a retornar `{ user, isNew: boolean }`. La detección de "nuevo" sucede cuando, después de buscar por `googleId` y por `email`, ninguno existe y se entra a la rama `prisma.user.create`.

El `User` model en `schema.prisma` actualmente **no tiene un campo `countryId`**. Verificar: la fase de onboarding necesita persistir el `countryId` seleccionado, pero no hay relación. Decisión locked: como el schema **no incluye** `countryId` en User, el campo se persiste solo si existe; si no, esta fase **solo persiste `acceptedTerms` como flag implícito** (no hay campo `acceptedTerms` tampoco). **Acción concreta:** este plan persiste el `countryId` como **dato del Profile** si existe campo, o **lo ignora pero valida la entrada** dejando un TODO para Phase 11 que añadirá `countryId` + `acceptedTerms` al modelo User. Para esta fase: validar `countryId` referenciado existe en `Country` (FK check manual) y validar `acceptedTerms === true`; no persistir en User (no hay campo). El JWT definitivo se emite igualmente — esto permite que el frontend continúe el flujo. Documentar este TODO en comentarios del método.

### Task 3: Modificar `AuthService` — upsert con isNew, helpers onboarding, googleOnboarding

**File:** `apps/api/src/auth/auth.service.ts`

**3.1 — Modificar `upsertGoogleUser()` (líneas 102-118):**

Cambiar firma a `Promise<{ user: User; isNew: boolean }>`. Lógica:
- Si encontrado por `googleId` → `{ user, isNew: false }`
- Si encontrado por `email` (linking) → `{ user (con googleId actualizado), isNew: false }` (el usuario YA existía, solo añadimos provider)
- Si creado nuevo → `{ user, isNew: true }` (mantener `sendWelcome` y creación de profile)

```ts
private async upsertGoogleUser(
  googleId: string,
  email: string,
  givenName?: string,
  familyName?: string,
): Promise<{ user: User; isNew: boolean }> {
  let user = await this.prisma.user.findUnique({ where: { googleId } });
  if (user) return { user, isNew: false };

  user = await this.prisma.user.findUnique({ where: { email } });
  if (user) {
    user = await this.prisma.user.update({ where: { id: user.id }, data: { googleId } });
    return { user, isNew: false };
  }

  user = await this.prisma.user.create({
    data: {
      email,
      googleId,
      firstname: givenName ?? null,
      lastname: familyName ?? null,
      role: 'AUTHENTICATED',
      confirmed: true,
    },
  });
  const slug = await this.generateProfileSlug(user.firstname, user.lastname, user.id);
  await this.prisma.profile.create({ data: { userId: user.id, slug } });
  await this.mail.sendWelcome(user.email, user.firstname ?? user.email);
  return { user, isNew: true };
}
```

**3.3 — Modificar `googleAuth()` (líneas 120-135):**

```ts
async googleAuth(accessToken: string) {
  const res = await fetch('https://www.googleapis.com/oauth2/v3/userinfo', {
    headers: { Authorization: `Bearer ${accessToken}` },
  });
  if (!res.ok) throw new UnauthorizedException('Token de Google inválido');
  const payload = await res.json() as {
    sub: string; email: string; email_verified: boolean;
    given_name?: string; family_name?: string;
  };
  if (!payload.email_verified) throw new UnauthorizedException('El email de Google no está verificado');
  if (!payload.email) throw new UnauthorizedException('Google no proporcionó un email');

  const { user, isNew } = await this.upsertGoogleUser(payload.sub, payload.email, payload.given_name, payload.family_name);
  if (user.blocked) throw new UnauthorizedException('Tu cuenta está bloqueada');
  if (isNew) {
    return { onboardingToken: this.signOnboardingPending(user), onboardingRequired: true as const };
  }
  return { token: this.sign(user), user: this.sanitize(user) };
}
```

**3.4 — Modificar `googleOneTap()` (líneas 137-153) con la misma lógica:**

```ts
async googleOneTap(credential: string) {
  const clientId = this.config.get<string>('GOOGLE_CLIENT_ID');
  const client = new OAuth2Client(clientId);
  let sub: string, email: string, givenName: string | undefined, familyName: string | undefined;
  try {
    const ticket = await client.verifyIdToken({ idToken: credential, audience: clientId });
    const p = ticket.getPayload();
    if (!p || !p.email_verified || !p.email) throw new Error();
    sub = p.sub; email = p.email; givenName = p.given_name; familyName = p.family_name;
  } catch {
    throw new UnauthorizedException('Credencial de Google inválida');
  }

  const { user, isNew } = await this.upsertGoogleUser(sub, email, givenName, familyName);
  if (user.blocked) throw new UnauthorizedException('Tu cuenta está bloqueada');
  if (isNew) {
    return { onboardingToken: this.signOnboardingPending(user), onboardingRequired: true as const };
  }
  return { token: this.sign(user), user: this.sanitize(user) };
}
```

**3.5 — Agregar `googleOnboarding()` (al final de la clase):**

```ts
/**
 * Completa el onboarding de un usuario Google nuevo: valida país y T&C, emite JWT definitivo.
 * El userId viene del OnboardingGuard (claim onboardingPending del JWT).
 *
 * TODO Phase 11: persistir countryId y acceptedTerms en el modelo User (requiere migración).
 * Por ahora solo se valida la entrada y se emite el JWT — el frontend asume completitud.
 */
async googleOnboarding(userId: number, countryId: number, acceptedTerms: boolean) {
  if (acceptedTerms !== true) {
    throw new BadRequestException('Debes aceptar los Términos y Condiciones');
  }
  const country = await this.prisma.country.findUnique({ where: { id: countryId } });
  if (!country) throw new BadRequestException('País no válido');

  const user = await this.prisma.user.findUnique({ where: { id: userId } });
  if (!user) throw new UnauthorizedException();
  if (user.blocked) throw new UnauthorizedException('Tu cuenta está bloqueada');

  return { token: this.sign(user), user: this.sanitize(user) };
}
```
