<objective>
Implementar **CNT-02: Favoritos** — endpoints `POST /events/:id/save` y `DELETE /events/:id/save` para guardar/desguardar eventos (D-07), `GET /users/me/saved-events` para listar favoritos del usuario autenticado (D-08), y campo derivado `isSaved` en respuestas de eventos cuando hay sesión (D-09) usando join eficiente sin N+1.

**Purpose:** Habilita el sistema de favoritos del usuario. UI consume `/cuenta/favoritos` (Design Brief). Patrón estándar de "save/unsave" que Twitter/Instagram aplican.

**Output:**
- `EventsService.save(eventId, userId)` y `unsave(eventId, userId)` con manejo de 409/404
- `EventsService.findSavedByUser(userId, page, limit)` lista paginada
- `EventsService.findAll(query, user?)` extendido con `isSaved` cuando user presente (batch query)
- `EventsService.findBySlug(slug, user?)` extendido con `isSaved` (findUnique sobre composite)
- `EventsController` con 2 endpoints nuevos (save/unsave) protegidos con JwtAuthGuard
- `UsersService.findSavedEventsForUser(userId, page, limit)` delega a EventsService
- `UsersController` con `GET /users/me/saved-events` autenticado
