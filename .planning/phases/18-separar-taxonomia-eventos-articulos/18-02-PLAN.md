<objective>
Migrar todo el backend (NestJS) para leer y escribir de los nuevos modelos `EventCategory`, `EventTag`, `ArticleCategory`, `ArticleTag`. Crear endpoints REST nuevos (`/event-categories`, `/event-tags`, `/article-categories`, `/article-tags`) y mantener `/categories` y `/tags` como aliases hacia los nuevos modelos para backwards compatibility durante el deploy. Actualizar TODOS los includes/selects/filtros/create/update en `events.service.ts`, `articles.service.ts`, `heroes.service.ts` y, lo más crítico, **OrdersService** (4 touchpoints) para usar el nuevo `eventCategory`.

Purpose: Sin esto, GET /events seguiría devolviendo `category` legacy y los servicios escribirían en la FK vieja. La crítica es OrdersService — si los 4 sitios no se actualizan, el cálculo de precios (`unitPrice * days`) devuelve `NaN` o `0`.

Output: Endpoints nuevos respondiendo; payloads de eventos/heroes con `eventCategory`; payload de artículos con `articleTags`; órdenes EVENT con `unitPrice > 0` real; alias `/categories` y `/tags` siguen vivos para que el frontend (Plan 18-03) pueda migrar en su propia wave.
</objective>

eventCategories() {
  return this.prisma.eventCategory.findMany({ orderBy: [{ order: 'asc' }, { name: 'asc' }] });
}
async findEventCategory(id: number) {
  const c = await this.prisma.eventCategory.findUnique({ where: { id } });
  if (!c) throw new NotFoundException('Categoría de evento no encontrada');
  return c;
}
async createEventCategory(dto: CreateEventCategoryDto) {
  await this.assertUniqueSlugFor('eventCategory', dto.slug);
  return this.prisma.eventCategory.create({ data: dto });
}
async updateEventCategory(id: number, dto: UpdateEventCategoryDto) {
  await this.findEventCategory(id);
  if (dto.slug) await this.assertUniqueSlugFor('eventCategory', dto.slug, id);
  return this.prisma.eventCategory.update({ where: { id }, data: dto });
}
async removeEventCategory(id: number) {
  await this.findEventCategory(id);
  await this.prisma.eventCategory.delete({ where: { id } });
  return { deleted: true };
}
