<objective>
Migrar todo el frontend (Next.js website) para consumir los nuevos endpoints `/event-categories` y `/article-tags`. Añadir tipos nuevos `ApiEventCategory`, `ApiEventTag`, `ApiArticleCategory`, `ApiArticleTag`; actualizar mappers (`toEventItem`, `toHeroSlide`) para leer `eventCategory` con fallback a `category` legacy; actualizar Header, sitemap, layout, página de categoría, búsqueda, formulario público `/crear/1`, EventForm del dashboard y ArticleForm del dashboard.

Purpose: El backend (plan 18-02) sigue devolviendo ambos campos (`category` y `eventCategory`) durante la transición. Este plan hace que el frontend prefiera siempre el nuevo y envíe los nuevos IDs en POST/PATCH. Después de este plan el frontend ya no depende de `category` legacy, lo que habilita al plan 18-04 a eliminar las tablas viejas sin riesgo.

Output: UI funcional con tipos correctos; todos los formularios envían los nombres nuevos de campos; consumidores legacy (otros archivos no listados) siguen funcionando porque exportamos aliases.
</objective>
