**NOTA CRÍTICA sobre greps de cleanup (acceptance criteria):**
Los identificadores legacy son substrings de los nuevos. Para evitar falsos positivos:
- `grep -c "category: true"` matchea también `eventCategory: true` → usar `grep -cP "(?<!event)category: true"` o `grep -v "eventCategory" | grep -c "category: true"`.
- `grep -c "categoryId Int?"` matchea también `eventCategoryId Int?` → usar `grep -cP "(?<!\\w)categoryId Int\\?"` (negative lookbehind para char alfanumérico).
- `grep -c "tags: true"` matchea también `articleTags: true` → usar `grep -cP "(?<!article)tags: true"`.
- `grep -c "tagIds"` matchea también `articleTagIds`/`eventTagIds` → usar `grep -cP "(?<!\\w)tagIds"`.
- `grep -c "category"` en general matchea `eventCategory`/`articleCategory` → usar `grep -cP "(?<!\\w)category(?!\\w)"` o anclar con espacios `" category"`.

Estos patterns están aplicados en los acceptance criteria abajo.
