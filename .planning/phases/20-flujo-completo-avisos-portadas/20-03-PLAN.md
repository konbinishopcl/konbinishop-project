<objective>
Fix the spot and hero creation forms in `CreateProductView.tsx` (route
`/crear-producto/spot` and `/crear-producto/hero`). The current form is broken in
five ways:
1. It sends `description`, `buttonText`, and `days` to POST /spots — none are DB
   columns; `days` in the create body means the item NEVER reaches the cart.
2. No image upload at all (static upload-box div).
3. No per-field validation — only a single toast for empty title.
4. It has a 4th "URL interna" linkType pill the backend enum does not support.
5. Hero form sends nothing as `titleAccent` (the state is `heroSubtitle` but it is
   never serialized) and allows submit without an image (backend requires it).

This plan rewrites BOTH form branches (spot + hero) to:
- Upload the image via `api.uploadImage` (hidden file input triggered by upload-box click), store the returned URL, show a preview.
- Validate with Zod, rendering errors as `<p className="field-error">` below each field.
- Map UI linkType `url|email|tel` → backend `URL|EMAIL|PHONE`; remove the `internal` pill.
- Submit flow: upload → POST create (NO days) → GET draft → PUT add-item (WITH days) → redirect /carrito.

The `articulo` branch is OUT OF SCOPE — leave it byte-for-byte unchanged.

Purpose: Make spot/hero creation actually work end-to-end into the cart.
Output: rewritten spot + hero branches of CreateProductView.tsx.

Requirements (defined inline for this phase):
- **SPOT-01:** POST /spots receives only { title, image?, linkType, linkValue } (no days/description/buttonText).
- **SPOT-02:** Spot image is uploaded and its URL stored before submit.
- **SPOT-03:** Spot fields validated with per-field error messages.
- **SPOT-04:** Spot days are forwarded to PUT /orders/:id/items, not the create call.
- **HERO-01..04:** Same four guarantees for heroes, plus titleAccent sent correctly and image required.
</objective>
