<objective>
Make the active organization a real piece of global state. Today `lib/api.ts` holds the
org id in a module-level variable (`_activeOrgId`) that resets on every page refresh, and
`UserMenu.tsx` keeps a local `activeOrg` state that dies when the menu unmounts. This plan
moves the active org into `UserContext` (the `useUser()` hook), persists it in
`localStorage` as `kb-org`, and re-applies it to `lib/api.ts` on app init so the
`X-Org-Context` header survives refreshes.

This is the foundation plan. It only touches `providers.tsx`. Plan 22-02 wires the visual
identity switch on top of the context this plan exposes.

Purpose: A user who switches to an organization stays "operating as" that org until they
explicitly switch back or log out — even after reloading the page.
Output: `OrgEntry` type + `activeOrg` / `setActiveOrg` exported from `useUser()`, with
localStorage persistence and `lib/api.ts` synchronization.
</objective>
