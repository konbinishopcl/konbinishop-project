<objective>
Turn the org context (delivered by Plan 22-01) into a visible identity switch. Today the
"Operando como" section only updates inside the open `UserMenu`; the navbar avatar and the
`/cuenta` sidebar keep showing the personal user. This plan:

1. Rewires `UserMenu.tsx` to read/write `activeOrg` from `useUser()` (Plan 22-01) instead
   of local component state, so a switch sticks after the menu closes and after a refresh.
2. Makes the navbar avatar (the `<button className="avatar">` that IS the menu trigger,
   inside `UserMenu.tsx`) show the org initial + purple gradient + org name title when an
   org is active.
3. Adds an "OPERANDO COMO" badge and org-styled avatar to the `/cuenta` sidebar
   (`AccountShell.tsx`), matching the design reference.

The "Mis eventos / Mis avisos / Mis portadas show org content" and "creation is attributed
to the org" requirements (ORG-05, ORG-06) need NO component changes — those pages already
call `api.myEvents/mySpots/myHeroes` and the create endpoints through `lib/api.ts`, which
injects `X-Org-Context` automatically once Plan 22-01 restores the context. This plan
VERIFIES that end-to-end in the human-verify checkpoint; it does not re-implement it.

Purpose: One switch = full identity change across the navbar and account area, persistent
across refresh.
Output: Context-driven `UserMenu` + org-aware `AccountShell`, verified end-to-end.
</objective>
