    En la función `uploadImage` (línea ~383), construir los headers de forma explícita para inyectar `X-Org-Context` SIN usar `buildHeaders()`. Razón (D-5): un upload multipart NO debe fijar `Content-Type` manualmente — el browser lo establece con el boundary; `buildHeaders()` fuerza `Content-Type: application/json`, lo que rompería el upload.

    Reemplazar el bloque actual:
    ```ts
    res = await fetch(`${apiBase()}/upload`, {
      method: "POST",
      headers: { Authorization: `Bearer ${token}` },
      body: form,
    });
    ```
    por:
    ```ts
    const headers: Record<string, string> = { Authorization: `Bearer ${token}` };
    if (_activeOrgId) headers["X-Org-Context"] = String(_activeOrgId);
    res = await fetch(`${apiBase()}/upload`, {
      method: "POST",
      headers,
      body: form,
    });
    ```
    `_activeOrgId` es la variable a nivel de módulo (línea 13), accesible directamente dentro de uploadImage. NO agregar `Content-Type` al objeto headers.
