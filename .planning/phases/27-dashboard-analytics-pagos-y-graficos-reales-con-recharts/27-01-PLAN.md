ApiPayment (what the JSON response items must look like):
```
{
  id: number;                    // Order.id
  status: "PAID" | "FAILED";     // Order.status (only these two are returned)
  total: number;                 // Order.total — WHOLE PESOS, not cents (confirmed: settings store pesos)
  gateway: string | null;        // Order.gateway (e.g. "TRANSBANK")
  createdAt: string;             // Order.createdAt ISO string
  buyer: {
    name: string;                // org name if orgId present, else owner name (see name rule below)
    handle: string | null;       // User.handle of org (if orgId) or owner
    email: string;               // User.email of org (if orgId) or owner
  };
  items: Array<{
    type: string;                // OrderItem.type (EVENT | SPOT | HERO | ARTICLE | SUBSCRIPTION)
    title: string;               // resolved server-side from the related record (see title rule)
    days: number;                // OrderItem.days
    subtotal: number;            // OrderItem.subtotal — WHOLE PESOS
  }>;
}
```

Prisma facts (from apps/api/prisma/schema.prisma):
- Order: id, status (OrderStatus), total Int, gateway String?, externalId String?, userId Int, orgId Int?, createdAt, items OrderItem[]
- Order relations: `owner` (User via userId), `org` (User? via orgId), `items` (OrderItem[])
- OrderItem: type (OrderItemType), days Int, unitPrice Int, subtotal Int, eventId?, spotId?, heroId?, articleId?
- OrderItem relations: event, spot, hero, article (all nullable)
- User has NO single `name` field — it has firstname String?, lastname String?, email String (unique), handle String? (unique), type UserType
- OrderStatus enum values: DRAFT, PENDING_PAYMENT, PAID, FAILED (confirmed in api.ts line 423)

User name rule (no `name` column exists):
  `[firstname, lastname].filter(Boolean).join(" ") || email`

Item title rule (resolve from whichever relation is non-null):
  - EVENT → item.event?.title
  - SPOT → item.spot?.title
  - HERO → item.hero?.titleAccent ?? item.hero?.title  (check actual field; fall back to type label)
  - ARTICLE → item.article?.title
  - else → the type string itself
  If the relation is null, use the type string (e.g. "EVENT") as title.

    Implementation:
    ```ts
    async findAllForAdmin() {
      const orders = await this.prisma.order.findMany({
        where: { status: { in: [OrderStatus.PAID, OrderStatus.FAILED] } },
        orderBy: { createdAt: 'desc' },
        include: {
          owner: { select: { firstname: true, lastname: true, email: true, handle: true } },
          org:   { select: { firstname: true, lastname: true, email: true, handle: true } },
          items: {
            include: {
              event:   { select: { title: true } },
              spot:    { select: { title: true } },
              hero:    { select: { title: true } },
              article: { select: { title: true } },
            },
          },
        },
      });

      const userName = (u: { firstname: string | null; lastname: string | null; email: string }) =>
        [u.firstname, u.lastname].filter(Boolean).join(' ') || u.email;

      return orders.map((o) => {
        const principal = o.org ?? o.owner;
        return {
          id: o.id,
          status: o.status as 'PAID' | 'FAILED',
          total: o.total,
          gateway: o.gateway,
          createdAt: o.createdAt.toISOString(),
          buyer: {
            name: userName(principal),
            handle: principal.handle,
            email: principal.email,
          },
          items: o.items.map((it) => ({
            type: it.type,
            title:
              it.event?.title ??
              it.spot?.title ??
              it.hero?.title ??
              it.article?.title ??
              it.type,
            days: it.days,
            subtotal: it.subtotal,
          })),
        };
      });
    }
    ```

    IMPORTANT: Before writing, open schema.prisma and confirm the exact title field names on Spot, Hero, Article, Event models. The Hero model may use `titleAccent` or `title` — use the real field. If a model has no `title`-like field, fall back to the type string only (do NOT invent fields). Adjust the `select` and the `??` chain to the real field names.
