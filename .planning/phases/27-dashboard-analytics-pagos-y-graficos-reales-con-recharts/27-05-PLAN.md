<interfaces>
From @/lib/api (Plan 02):
- api.adminPayments(token) -> Promise ApiPayment[]
- ApiPayment = { id; status: "PAID"|"FAILED"; total: number; gateway; createdAt: string (ISO); buyer; items }
From @/components/providers: useUser() -> { token }
From @/components/charts/RevenueBarChart (Plan 02):
- import { RevenueBarChart, type RevenueDatum } from "@/components/charts/RevenueBarChart"
- Props: { data: { label: string; value: number }[] }; renders .empty panel when data.length === 0.

Period bucketing spec (client-side; only PAID payments count toward revenue):
- "Día":    last 7 days. 7 buckets, one per day. label = day-of-month or "L/M/X..." weekday initial; use `new Date(p.createdAt).getDate()` grouping over the last 7 calendar days, label = two-digit day.
- "Semana": last 8 weeks. 8 buckets, one per ISO-ish week (group by floor(diffDays/7)). label = "S1".."S8" oldest→newest.
- "Mes":    last 12 months. 12 buckets, one per month. label = month initial ["E","F","M","A","M","J","J","A","S","O","N","D"] aligned to the actual months, oldest→newest.
- "Año":    last 5 years. 5 buckets, one per year. label = 2-digit year.
Each bucket value = sum of `total` of PAID payments whose createdAt falls in that bucket. Buckets with zero stay in the array with value 0 (so the chart shows the time axis), EXCEPT: if ALL buckets are 0 (no PAID payments anywhere in range), pass an empty array to RevenueBarChart so it renders the empty-state panel.
</interfaces>
