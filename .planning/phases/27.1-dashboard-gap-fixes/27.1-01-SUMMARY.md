Nine token-authenticated create/update/delete methods added to `api.ts` for countries, states, and cities — enabling Plan 07 to wire `SimpleCRUDSection` to the real backend catalog API.

## What Was Built

Added a "Geo admin CRUD (Phase 27.1)" section to the `api` object in `apps/website/lib/api.ts` with exactly 9 new methods:

- `createCountry(body, token)` / `updateCountry(id, body, token)` / `deleteCountry(id, token)`
- `createState(body, token)` / `updateState(id, body, token)` / `deleteState(id, token)`
- `createCity(body, token)` / `updateCity(id, body, token)` / `deleteCity(id, token)`

All methods use the existing `request<T>(path, options, token)` helper pattern. Delete methods return `{ deleted: boolean }` (not `void`) per the catalog service implementation.

The three existing read-only methods (`countries`, `regions`, `communes`) are untouched.
