# Phase 27.1 Plan 02: Dashboard Gap Fixes (Breadcrumb + SubsSection Settings) Summary

**One-liner:** Group-aware breadcrumb using `activeGroup` in DashboardShell; SubsSection prefills 4 plan fields from `/api/settings` on mount.

## What Was Built

Two independent dashboard fixes:

1. **DashboardShell breadcrumb** — Added `crumbPrefix` const that derives the breadcrumb prefix from the active nav group name instead of hardcoding "DASHBOARD". Sub-pages with their own crumb (Términos, Nuevo, Editar) retain their verbatim crumb string.

2. **SubsSection settings load** — Added a `useEffect` that runs on mount (and when `token` changes) to fetch `/api/settings`, filter the 4 plan keys, and patch the `plan` state so the form reflects previously saved values rather than hardcoded defaults.
