<objective>
Completar el frontend de Phase 28 en la capa de formularios y dashboard admin: agregar un multi-select
de categorías a `ArticleForm` (clon exacto del selector de tags ya existente — D-10), pre-seleccionar las
categorías del artículo en modo edit, enviar `articleCategoryIds: number[]` en el payload, y mostrar la
primera categoría + badge de conteo en `ArticlesSection` (D-11).

la lista admin.
Output: ArticleForm con selector de categorías, edit page mapeando categorías, ArticlesSection con display D-11.

Depende de Plan 04 (tipo `ApiArticle.articleCategories` + `ApiArticleCategory` exportados en lib/api.ts).
autonomous: false — incluye un checkpoint de verificación humana del selector de categorías (D-10 es
manual-only en 28-VALIDATION.md).
