## Component Inventory

### New: Category multi-select in `ArticleForm`

This is a direct clone of the existing tags selector. No new visual pattern is introduced.

**Visual spec:**
- Label: `Categorías` with count inline: `(N seleccionadas)` — same pattern as line 313 of `ArticleForm.tsx`
- Search input: placeholder `Buscar categoría…`
- Scrollable checkbox list: `max-height: 180px`, `overflow-y: auto`, `border: 1px solid var(--line)`, `border-radius: 8px`, `padding: 8px`
- Each row: `display: flex; align-items: center; gap: 8px; padding: 6px 8px; border-radius: 6px; font-size: 13px`
- Empty list fallback: `Sin categorías` — centered, `color: var(--ink-3); font-size: 12px; padding: 12px`
- Applies to: admin create, admin edit, organizer create (`variant="sponsored"` via `/crear-articulo`), organizer edit. All flows use the same `ArticleForm` component.

### Modified: Category display in `ArticleCard`, `ArticleView`, `NoticiasHubView`, `SearchLightbox`

**Visual contract (D-09):** Show `articleCategories[0].name` (primary category only). No multi-badge. No visual change to existing category label elements.

- Display element: existing `.a-cat` class in `ArticleCard` — no style change
- Fallback when `articleCategories` is empty: text `"Noticias"` (matches existing fallback in `getCat()` returning `"Noticias"`)
- Helper function name: `primaryCategory(a: ApiArticle): string` — replaces inline `getCat()` in `ArticleCard.tsx` and `ArticleView.tsx`

### Modified: Category display in `ArticlesSection` (admin dashboard list)

**Visual contract (D-11):** Show first category name + count badge when multiple categories exist.

- Format: `{firstCategoryName} +{remaining}` — example: `Anime +2`
- Element: existing `.pill` class (background `--surface-2`, color `--ink-2`, border `--line`) — no new styles needed
- When zero categories: show nothing (empty cell) — consistent with existing missing-field behavior
- When one category: show name only, no `+0` suffix

## States and Interactions

### Category multi-select — interaction contract

| State | Visual |
|-------|--------|
| Unchecked | Native checkbox unchecked + tag name in `--ink` |
| Checked | Native checkbox checked + tag name in `--ink` |
| Search filter active | List filters in real-time; no animation |
| Loading (fetch in flight) | List renders empty; `Sin categorías` copy shown until data resolves |
| Error loading categories | `Sin categorías` copy shown (silent failure, same as tag fetch fallback) |

Fetch triggered on mount via `useEffect` → `fetch("/api/article-categories")`. No loading spinner — same pattern as existing tag fetch.

### ArticleForm — edit mode category pre-selection

When `initial.articleCategories` is provided, `selectedCategoryIds` is initialized from `initial.articleCategories.map(c => c.id)`. Checkboxes render pre-checked on mount.

### NoticiasHubView — rail grouping (internal logic)

Category rails group by `articleCategories?.[0]?.slug` (first category only, D-09). No visual change to the rails themselves.
