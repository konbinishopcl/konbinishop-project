# Phase 28.1 Plan 01: Real Author Name in Article Detail Summary

**One-liner:** Backend ARTICLE_DETAIL_INCLUDE extended with user select; ApiArticle typed with optional user field; ArticleView derives real author name with "Konbini Editorial" fallback for null/blank-name cases.

## What Was Built

### Task 1 — Backend: ARTICLE_DETAIL_INCLUDE + user

Added `user: { select: { firstname: true, lastname: true } }` to `ARTICLE_DETAIL_INCLUDE` in `articles.service.ts`. The existing `ARTICLE_INCLUDE` already had user (with email too), but `ARTICLE_DETAIL_INCLUDE` was missing it entirely.

### Task 2 — Type: ApiArticle.user field

Added `user?: { firstname: string; lastname: string } | null` to `ApiArticle` in `lib/api.ts`. Optional (`?`) for backwards compatibility with consumers that don't need author info; `| null` covers articles with no associated user (WordPress imports).

### Task 3 — Frontend: Dynamic authorName

Replaced `const authorName = "Konbini Editorial"` with an IIFE that:
1. Returns "Konbini Editorial" when `article.user` is null/undefined (WP-imported articles)
2. Returns "Konbini Editorial" when firstname + lastname trim to empty string
3. Returns `"${firstname} ${lastname}".trim()` for articles with real user data

The `authorInitials()` call and display template required zero changes — they already use the `authorName` const.

The caption string `"FOTO · {cat.toUpperCase()} · KONBINI EDITORIAL"` was intentionally left unchanged per plan scope boundary.
