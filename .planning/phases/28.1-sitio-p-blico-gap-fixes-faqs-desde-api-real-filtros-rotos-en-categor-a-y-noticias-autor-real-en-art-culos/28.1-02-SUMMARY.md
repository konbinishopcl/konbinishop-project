# Phase 28.1 Plan 02: FAQ Pages Wired to API Summary

## One-liner

Both `/preguntas-frecuentes` and `/precios` FAQ sections wired to `api.faqAll()` as async SSR with normalized `ApiFaqItem` object shape and graceful fallback to hardcoded arrays.

## What Was Built

### Task 1: Wire /preguntas-frecuentes to api.faqAll()
- Converted `PreguntasFrecuentesPage` to `async` server component
- Added `api.faqAll()` call with empty-array guard; falls back to `FAQS_FALLBACK` tuples converted to `ApiFaqItem` objects
- Eliminated tuple indexing (`q[0]`/`q[1]`); render now uses `item.question`/`item.answer` consistently

### Task 2: Wire /precios FAQ section to api.faqAll()
- `precios/page.tsx`: added separate FAQ fetch outside existing `Promise.all` with `FAQS_FALLBACK` constant and `ApiFaqItem` normalization
- `PricingView.tsx`: removed module-level `FAQS` constant; added `faqs: ApiFaqItem[]` to `Props` type; destructured from function signature; render uses `item.question`/`item.answer`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] ApiFaqItem includes createdAt/updatedAt fields not in plan template**
- **Found during:** Task 1 build verification
- **Issue:** Plan showed fallback as `{ id, question, answer, order }` but `ApiFaqItem` type in `api.ts` also requires `createdAt: string` and `updatedAt: string`
- **Fix:** Added `createdAt: ""` and `updatedAt: ""` to all fallback object literals in both pages
- **Files modified:** `preguntas-frecuentes/page.tsx`, `precios/page.tsx`
- **Commit:** 229d740 (included in Task 2 commit)

## Known Stubs

None — all FAQ data is served from the API when available, with real hardcoded fallback content for offline scenarios.
