<decisions>
- Profile model holds the Strapi User extras (rut, isCompany, firstname, lastname, role), PK = Neon Auth user id (String).
- User relations on Event become plain String columns (userId, approvedById, rejectedById) — Neon Auth is the source of truth, no Prisma FK.
- Strapi `media` fields -> String (single) or String[] (gallery): store URLs.
- Strapi `date` -> DateTime @db.Date; Strapi `time` -> String ("HH:mm").
- Strapi `uid` (slug) -> unique String.
- m2m relations use Prisma implicit many-to-many.
- approved_by corrected to n:1 (one approver per event).
- The neon_auth.users_sync table is NOT mapped yet (Neon Auth not activated); deferred follow-up.
</decisions>
