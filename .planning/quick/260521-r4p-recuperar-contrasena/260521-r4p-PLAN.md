# Quick Task r4p: Endpoints de recuperación de contraseña

**One-liner:** Agregar a la API el flujo de recuperación de contraseña — solicitar un token y
restablecer la contraseña con él.

## Tareas

1. **Schema** — agregar `resetToken` y `resetTokenExpiry` al modelo `User` + migración.
2. **DTOs** — `ForgotPasswordDto` (email) y `ResetPasswordDto` (token + password).
3. **AuthService** — `forgotPassword(email)` y `resetPassword(token, password)`.
4. **AuthController** — `POST /api/auth/forgot-password` y `POST /api/auth/reset-password`.

## Decisiones

- El token se guarda hasheado (SHA-256); validez 1 hora.
- Sin infraestructura de email aún: el token se registra en el log del servidor
  (`console.log`). En v2 se enviará por correo.
- `forgot-password` responde siempre `{ ok: true }` — no revela si el email existe.

## Verificación

- `nest build` compila sin errores
- `forgot-password` genera el token; `reset-password` con un token válido cambia la
  contraseña; con un token inválido devuelve `400`
