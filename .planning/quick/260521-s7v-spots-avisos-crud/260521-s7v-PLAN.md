# Quick Task s7v: Feature de avisos (Spots) en la API

**One-liner:** Convertir el modelo `Spot` en el feature de avisos pagados — CRUD completo en
la API, sin pasarela de pagos por ahora.

## Contexto

Los "spots" son avisos pagados que se muestran entre las cards de eventos: un título con una
imagen y/o un contacto, sin vista de detalle. El link va a una URL, un teléfono o un email.
Cualquier usuario autenticado puede crear uno; queda activo al crearse (no hay pagos aún).
El nombre `Spot` se mantiene a propósito (los ad-blockers bloquean rutas con "ad").

## Tareas

1. **Schema** — rehacer el modelo `Spot`: `title` (requerido), `image?`, `linkType` (enum
   `SpotLinkType`: URL/PHONE/EMAIL), `linkValue`, `expirationDate?`, `owner` (User). Migración.
2. **Seed** — actualizar la sección de spots al nuevo shape (creada tras el usuario dueño).
3. **Módulo `spots`** — controller + service + DTOs:
   - `GET /api/spots` (público, solo vigentes)
   - `GET /api/spots/mine` (autenticado)
   - `POST /api/spots` (cualquier autenticado)
   - `PATCH /api/spots/:id` y `DELETE /api/spots/:id` (dueño o admin)
4. **Catalog** — quitar el `GET /spots` que tenía (se mueve al módulo nuevo).

## Decisiones

- Nombre `Spot` se mantiene (anti ad-blocker).
- Activo al crearse — sin paso de pago (la pasarela es trabajo futuro).
- Código, comentarios y docs en inglés; solo los mensajes al usuario final en español.

## Verificación

- `nest build` compila sin errores
- Crear / listar / editar / borrar spots funciona; `linkType` inválido → 400; sin sesión → 401
