## Tech Debt Summary

### CategoryView filter bugs (3 items — same root cause)
`filterFormat`, `filterQuick`, and `dateFrom/dateTo` are tracked in state, shown in the filter bar, and incremented in `activeCount` — but no `filtered = filtered.filter(...)` call exists for them. User sees "Limpiar (1)" but the event list doesn't change. **Root cause:** `EventItem` type has no `format` field and no ISO date; filters cannot be implemented without a mapper change.

### Dead component (1 item)
`/noticias/categoria/[slug]/NewsCategoryView.tsx` (489 lines) is unreachable — its page.tsx is a redirect shim. The file still has `TYPE_OPTIONS` that was cleaned elsewhere in 28.1.
